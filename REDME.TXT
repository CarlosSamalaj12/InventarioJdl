README — Sistema de Bodega / Inventario (MariaDB)
1) ¿Qué es este sistema?

Este sistema es una base de datos para manejar inventario tipo Kardex, con:

Bodegas (principal / receptora / operativas)

Productos con categorías, medidas y proveedores

Movimientos (entradas, salidas, transferencias, ajustes)

Kardex como historial que calcula el stock real

Pedidos (solicitudes) desde bodegas operativas hacia la bodega surtidora (normalmente la principal)

3) Tipos de bodega (cómo interpretarlo)
Bodega PRINCIPAL

Es la bodega “madre”.

Normalmente concentra el stock y surte pedidos de otras bodegas.

Puede recibir compras y despachar a operativas.

Bodega RECEPTORA

Es la “puerta de entrada” (recepción).

Puede existir si querés separar “recibo” vs “almacenaje”.

En tu script está prevista por el ENUM, pero vos decidís si la usás en la operación. 

script bodega

Bodega OPERATIVA

Es la bodega del día a día (Cocina, Restaurante, Camarería).

Puede manejar stock propio y recibir transferencias desde la principal. 

script bodega

El script incluye triggers para que NO se puedan editar ni eliminar movimientos de fechas pasadas.

Bloquea update/delete de movimiento_encabezado

10) Flujo típico de operación (ejemplos)
A) Compra / Entrada

Crear movimiento_encabezado tipo ENTRADA con motivo Compra, proveedor y documento.

Agregar líneas en movimiento_detalle (producto, lote, vencimiento, cantidad, costo).

Confirmar: se refleja en kardex y sube el stock.

B) Consumo / Salida

Crear movimiento tipo SALIDA (motivo Consumo Interno).

Se descuenta en kardex (delta negativo).

C) Transferencia Principal → Cocina

Movimiento tipo TRANSFERENCIA con bodega origen=Principal y destino=Cocina.

Kardex registra: sale de Principal (–) y entra a Cocina (+).

D) Pedido Cocina → Principal

Cocina crea pedido (PENDIENTE).

Bodeguero aprueba.

Al surtir, el sistema genera el movimiento (normalmente transferencia o salida según tu regla) y lo vincula al pedido.
